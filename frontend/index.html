IPC Concepts

Five fundamental IPC mechanisms every OS student must master

🔧
POSIX · Unix

Pipes

Unidirectional data channels connecting two processes. Named pipes (FIFOs) extend this to unrelated processes, enabling linear producer-consumer flows with kernel-buffered I/O.

Complexity: Low →
📬
POSIX · SysV

Message Queues

Kernel-maintained queues that store typed messages. Processes send and receive discrete message blocks asynchronously, decoupling producers from consumers with priority support.

Complexity: Medium →
💾
mmap · SysV

Shared Memory

The fastest IPC mechanism — a memory region mapped into multiple process address spaces. Requires explicit synchronization (semaphores/mutexes) to prevent race conditions and data corruption.

Complexity: High →
🔒
Dijkstra · POSIX

Semaphores

Integer-valued signaling primitives using atomic wait() and signal() (P/V) operations. Binary semaphores act as mutexes; counting semaphores manage finite resource pools across processes.

Complexity: Medium →
🌐
BSD · POSIX

Sockets

Bidirectional endpoints enabling IPC both locally (Unix domain sockets) and across networks (TCP/UDP). The foundation of all networked applications and inter-host process communication.

Complexity: High →

IPC Workflow

Every IPC mechanism follows this five-step execution model

01
⚙️

Create Process

fork() spawns child process. OS allocates PCB, virtual address space, and file descriptor table.

pid = fork()
02
📤

Send Data

Process A writes to the IPC channel using the mechanism's write/send/msgsnd API call.

write(fd, buf, n)
03
🔄

Synchronize

Semaphores or mutexes enforce mutual exclusion where required. Critical section is protected.

sem_wait(&sem)
04
📥

Receive Response

Process B reads from the channel. Kernel handles data transfer, blocking if data unavailable.

read(fd, buf, n)
05
🖥️

Display Output

Simulator renders the result — data received, semaphore state, memory segment value — in the console.

printf(output)
